## 2. Function-Based View

View 是一個接收 `HttpRequest`、回傳 `HttpResponse` 的 Python 函式。

### 2-1. Homepage（純 HttpResponse）

```python
# pages/views.py
from django.http import HttpResponse
from django.shortcuts import render

def home_page_view(request):
    return HttpResponse("Homepage")
```

### 2-2. About page（使用 Template）

`render()` 是 Django 的 shortcut，參數順序：`render(request, template_name, context)`

```python
def about_page_view(request):
    context = {
        "name": "Alice",
        "age": 33,
    }
    return render(request, "pages/about.html", context)
```

## 3. URL 設定

### 3-1. App 層 URL（`pages/urls.py`）

```python
from django.urls import path
from .views import home_page_view, about_page_view

urlpatterns = [
    path("about/", about_page_view),
    path("", home_page_view),
]
```

### 3-2. 專案層 URL（`django_project/urls.py`）

```python
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path("admin/", admin.site.urls),
    path("", include("pages.urls")),  # new
]
```

## 4. Templates

Django 預設在每個 App 內找 `templates/` 子目錄。

為了避免不同 App 的同名 template 衝突，採用**命名空間**結構：

```
pages/templates/pages/about.html
  ↑       ↑      ↑
 app    固定    app name（namespace）
```

```html
<!-- pages/templates/pages/about.html -->
<h1>About page</h1>
<p>My name is {{ name }}. I am {{ age }} years old.</p>
```

### Template Context

- `context` 是一個 dict，key 對應到 template 裡的變數名稱
- template 用 `{{ 變數名 }}` 顯示變數

### Django Template Language 常用語法

| 語法 | 說明 |
|------|------|
| `{{ variable }}` | 顯示變數值 |
| `{% tag %}` | 執行邏輯（如迴圈、判斷） |
| `{% comment %}...{% endcomment %}` | 註解，不會顯示在頁面上 |

## 5. Testing

### 為什麼要寫測試？

> "Code without tests is broken as designed." — Jacob Kaplan-Moss（Django 共同創始人）

### Django 測試類別選擇

| 類別 | 使用時機 |
|------|----------|
| `SimpleTestCase` | 不需要資料庫 |
| `TestCase` | 需要資料庫 |
| `TransactionTestCase` | 需要測試資料庫 transaction |
| `LiveServerTestCase` | 需要真實瀏覽器（Selenium） |

### 本週測試範例

```python
# pages/tests.py
from django.test import SimpleTestCase

class HomepageTests(SimpleTestCase):
    def test_url_exists_at_correct_location(self):
        response = self.client.get("/")
        self.assertEqual(response.status_code, 200)

class AboutpageTests(SimpleTestCase):
    def test_url_exists_at_correct_location(self):
        response = self.client.get("/about/")
        self.assertEqual(response.status_code, 200)
```

```bash
python manage.py test
```

> 注意：URL 結尾要有 `/`，少了 `/` 會得到 301 redirect 而非 200！

## 重點整理

```
Django 完整 Request/Response 流程（含 Template）：

瀏覽器 Request
    ↓
urls.py（路由比對）
    ↓
views.py（處理邏輯、準備 context）
    ↓
templates/（渲染 HTML）
    ↓
HttpResponse（回傳給瀏覽器）
```

- **Template 命名空間**：`app/templates/app/template.html`，避免不同 App 同名衝突
- **`render()`**：Django shortcut，把 request、template、context 組合成 HttpResponse
- **Context**：dict 型態，把資料從 view 傳進 template
- **Unit test vs Integration test**：unit test 快且好維護，優先多寫；integration test 少量補充

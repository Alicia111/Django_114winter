## 2. 資料庫概念

### 關聯式 vs 非關聯式

| 種類 | 說明 | 範例 |
|------|------|------|
| 關聯式 (Relational) | 用表格（欄、列）儲存，透過 SQL 操作 | SQLite, PostgreSQL, MySQL |
| 非關聯式 (Non-relational) | 文件、Key-Value、Graph 等形式 | MongoDB, Redis |

Django 只內建支援**關聯式資料庫**。

### Django ORM

ORM (Object-Relational Mapper) 讓我們用 **Python 寫資料庫操作，不用手寫 SQL**。

Django ORM 支援：SQLite、PostgreSQL、MySQL、MariaDB、Oracle。

## 3. Database Model（資料庫模型）

```python
# posts/models.py
from django.db import models


class Post(models.Model):
    text = models.TextField()

    def __str__(self):
        return self.text[:50]  # 前 50 字當作顯示名稱
```

- 繼承 `models.Model` → Django 知道這是個資料庫模型
- `TextField()` → 適合存放長文字
- `__str__()` → 讓 Admin 顯示人讀得懂的名稱（最佳實踐，每個 Model 都要加）

### 常用 Field 類型

| Field | 說明 |
|-------|------|
| `CharField(max_length=200)` | 短文字，需設 max_length |
| `TextField()` | 長文字 |
| `IntegerField()` | 整數 |
| `DateTimeField()` | 日期時間 |
| `BooleanField()` | True/False |

## 4. Activating Models（啟用模型）

每次新增或修改 Model，都要執行兩步驟：

```bash
# 第一步：產生 migration 檔案（記錄變更）
python manage.py makemigrations posts

# 第二步：套用到資料庫
python manage.py migrate
```

```
Migrations for 'posts':
  posts/migrations/0001_initial.py
   - Create model Post
```

- `makemigrations` → 把 Model 的變更記錄成 migration 檔（可追蹤歷史）
- `migrate` → 實際執行 migration，更新資料庫結構

## 6. Django Shell（CLI 新增資料）

除了 Admin 介面，也可以用 **Django Shell** 直接透過 CLI 操作資料庫，適合快速測試或手動塞假資料。

```bash
python manage.py shell
```

進入互動式 Python shell 後，使用 ORM 操作：

```python
>>> from posts.models import Post

# 新增一筆資料
>>> Post.objects.create(text="Hello from shell!")

# 查看所有資料
>>> Post.objects.all()
<QuerySet [<Post: Hello from shell!>]>

# 離開
>>> exit()
```

語法跟 `tests.py` 裡的 `Post.objects.create(...)` 完全相同。

## 7. QuerySet 與 ORM 查詢

```python
Post.objects.all()
```

拆解說明：

| 部分 | 說明 |
|------|------|
| `Post` | 我們的 Model class（對應資料表） |
| `objects` | 預設的 Manager（每個 Model 都有）|
| `all()` | 回傳所有資料的 QuerySet |

QuerySet 是一個資料庫查詢結果的集合，可以進一步 filter、order 等。

## 8. Function-Based View（FBV）

```python
# posts/views.py
from django.shortcuts import render
from .models import Post


def post_list(request):
    posts = Post.objects.all()
    return render(request, "post_list.html", {"posts": posts})
```

## 10. ListView（Generic Class-Based View）

`ListView` 是 Django 內建的通用 CBV，專門用來列出資料庫所有資料。

```python
# posts/views.py
from django.views.generic import ListView
from .models import Post


class PostList(ListView):
    model = Post
    template_name = "post_list.html"
```

⚠️ 注意：ListView 預設把 context 變數命名為 `<model>_list`，所以要改 template：

```html
<!-- templates/post_list.html -->
<h1>Message board homepage</h1>
<ul>
  {% for post in post_list %}   ← 改成 post_list，不是 posts
  <li>{{ post.text }}</li>
  {% endfor %}
</ul>
```

```python
# posts/urls.py
from django.urls import path
from .views import PostList

urlpatterns = [
    path("", PostList.as_view(), name="home"),  # new
]
```

### FBV vs ListView 比較

| | FBV | ListView (CBV) |
|--|-----|----------------|
| context 變數 | 自己命名（`posts`）| 自動用 `<model>_list` |
| 程式碼量 | 多一點 | 極簡 |
| 彈性 | 較高 | 中等 |

## 11. Testing

本週開始用 **`TestCase`**（不是 `SimpleTestCase`），因為有資料庫了！

`TestCase` 會自動建立獨立的測試資料庫，跑完測試後銷毀，不影響實際 DB。

### setUpTestData()

```python
# posts/tests.py
from django.test import TestCase
from django.urls import reverse
from .models import Post


class PostTests(TestCase):
    @classmethod
    def setUpTestData(cls):
        cls.post = Post.objects.create(text="This is a test!")

    def test_model_content(self):
        self.assertEqual(self.post.text, "This is a test!")

    def test_url_exists_at_correct_location(self):
        response = self.client.get("/")
        self.assertEqual(response.status_code, 200)

    def test_homepage(self):           # 把多個 assertions 合在一起
        response = self.client.get(reverse("home"))
        self.assertEqual(response.status_code, 200)
        self.assertTemplateUsed(response, "post_list.html")
        self.assertContains(response, "This is a test!")
```

### 新的測試觀念

| 觀念 | 說明 |
|------|------|
| `TestCase` | 有資料庫的測試（vs `SimpleTestCase` 無資料庫）|
| `@classmethod` + `setUpTestData(cls)` | 整個 test class 只建立一次測試資料，效能較好 |
| `cls.post` | 在測試方法中用 `self.post` 存取 |
| `test_homepage` | 把多個相關 assertion 合在一個 test 裡，簡潔易讀 |

```bash
python manage.py test

# Creating test database for alias 'default'...
# Ran 3 tests in 0.005s
# OK
# Destroying test database for alias 'default'...
```

## 重點整理

```
資料庫流程：
  Model 定義 → makemigrations → migrate → Admin 註冊

QuerySet：
  Post.objects.all()  → 取所有資料
  Post.objects.create(text="...")  → 建立資料

FBV vs ListView：
  FBV  → 自己寫 context，命名自由
  ListView → model + template_name，context 自動叫 <model>_list

Testing 新知：
  SimpleTestCase → 無 DB
  TestCase       → 有 DB（自動建立/銷毀測試 DB）
  setUpTestData  → 整個 class 共用，效能優於 setUp
```

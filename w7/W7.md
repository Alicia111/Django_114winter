## 2. Database Model（含 ForeignKey）

```python
# blog/models.py
from django.db import models
from django.urls import reverse


class Post(models.Model):
    title = models.CharField(max_length=200)
    author = models.ForeignKey(
        "auth.User",
        on_delete=models.CASCADE,
    )
    body = models.TextField()

    def __str__(self):
        return self.title

    def get_absolute_url(self):
        return reverse("post_detail", kwargs={"pk": self.pk})
```

### ForeignKey

| 概念 | 說明 |
|------|------|
| `ForeignKey` | 一對多關聯。一個 User 可以寫很多 Post，每篇 Post 只屬於一個 User |
| `"auth.User"` | 用字串引用 Django 內建的 User Model（不用 import） |
| `on_delete=CASCADE` | 當作者被刪除時，他寫的 Post 也一併刪除 |

其他 `on_delete` 選項常用的還有：
- `PROTECT` → 禁止刪除作者（會丟錯）
- `SET_NULL` → 把 author 欄位設為 NULL（需 `null=True`）

## 3. get_absolute_url() 與 reverse()

```python
def get_absolute_url(self):
    return reverse("post_detail", kwargs={"pk": self.pk})
```

- **`reverse("post_detail", kwargs={"pk": self.pk})`**
  → 透過 URL name 產生實際路徑，例如 `/post/1/`
- **`get_absolute_url()`** 是 Django 約定俗成的方法名稱：
  - Template 可直接用 `{{ post.get_absolute_url }}` 取得連結
  - Admin 會自動加上「View on site」按鈕
  - 之後 CreateView/UpdateView 成功後會自動跳轉到這個 URL

→ 寫一次，到處都能用，避免 template 裡硬編碼路徑。

## 5. Django Admin 客製化（ModelAdmin）

W5 是直接 `admin.site.register(Post)`，這次進階一點：

```python
# blog/admin.py
from django.contrib import admin
from .models import Post


class PostAdmin(admin.ModelAdmin):
    list_display = (
        "title",
        "author",
        "body",
    )


admin.site.register(Post, PostAdmin)
```

- `list_display` → Admin 列表頁要顯示哪些欄位（不指定就只顯示 `__str__()`）
- 可以再加：`list_filter`、`search_fields`、`ordering` 等

```bash
python manage.py createsuperuser
python manage.py runserver
# → http://127.0.0.1:8000/admin/
```

## 6. URLs（含動態路由）

```python
# django_project/urls.py
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path("admin/", admin.site.urls),
    path("", include("blog.urls")),
]
```

```python
# blog/urls.py
from django.urls import path
from .views import post_list, post_detail

urlpatterns = [
    path("post/<int:pk>/", post_detail, name="post_detail"),  # 動態路由
    path("", post_list, name="home"),
]
```

### `<int:pk>` 拆解

| 部分 | 說明 |
|------|------|
| `<...>` | 路徑參數，會傳給 view function |
| `int:` | path converter，限制只能是整數 |
| `pk` | 變數名稱，view function 收到的參數名 |

→ `/post/3/` 會把 `pk=3` 傳進 `post_detail(request, pk)`

其他常用 converter：`str`（預設）、`slug`、`uuid`。

## 7. Views（兩個 FBV）

```python
# blog/views.py
from django.shortcuts import render, get_object_or_404
from .models import Post


def post_list(request):
    posts = Post.objects.all()
    return render(request, "home.html", {"posts": posts})


def post_detail(request, pk):
    post = get_object_or_404(Post, pk=pk)
    return render(request, "post_detail.html", {"post": post})
```

### get_object_or_404()

| 寫法 | 行為 |
|------|------|
| `Post.objects.get(pk=pk)` | 找不到 → 丟 `DoesNotExist` 例外（500 錯誤） |
| `get_object_or_404(Post, pk=pk)` | 找不到 → 自動回傳 **404 Not Found**（正確的使用者體驗）|

→ 詳情頁、編輯頁等只要是「依 ID 查單筆」的場景，幾乎都用 `get_object_or_404`。

## 10. Testing（測試 ForeignKey 與 DetailView）

```python
# blog/tests.py
from django.contrib.auth import get_user_model
from django.test import TestCase
from django.urls import reverse

from .models import Post


class BlogTests(TestCase):
    @classmethod
    def setUpTestData(cls):
        cls.user = get_user_model().objects.create_user(
            username="testuser", email="test@email.com", password="secret"
        )
        cls.post = Post.objects.create(
            title="A good title",
            body="Nice body content",
            author=cls.user,
        )

    def test_post_model(self):
        self.assertEqual(self.post.title, "A good title")
        self.assertEqual(self.post.body, "Nice body content")
        self.assertEqual(self.post.author.username, "testuser")
        self.assertEqual(str(self.post), "A good title")
        self.assertEqual(self.post.get_absolute_url(), "/post/1/")

    def test_url_exists_at_correct_location_listview(self):
        response = self.client.get("/")
        self.assertEqual(response.status_code, 200)

    def test_url_exists_at_correct_location_detailview(self):
        response = self.client.get("/post/1/")
        self.assertEqual(response.status_code, 200)

    def test_post_listview(self):
        response = self.client.get(reverse("home"))
        self.assertEqual(response.status_code, 200)
        self.assertContains(response, "Nice body content")
        self.assertTemplateUsed(response, "home.html")

    def test_post_detailview(self):
        response = self.client.get(
            reverse("post_detail", kwargs={"pk": self.post.pk})
        )
        no_response = self.client.get("/post/100000/")
        self.assertEqual(response.status_code, 200)
        self.assertEqual(no_response.status_code, 404)
        self.assertContains(response, "A good title")
        self.assertTemplateUsed(response, "post_detail.html")
```

### 本週新增的測試重點

| 寫法 | 說明 |
|------|------|
| `get_user_model()` | 取得目前的 User Model（比直接 import `User` 更安全，未來換 Custom User 不用改測試） |
| `create_user(...)` | 建立測試用使用者（密碼會自動加密） |
| 同時測 `200` 與 `404` | DetailView 要驗證「存在的 pk → 200」與「不存在的 pk → 404」 |
| `reverse("post_detail", kwargs={"pk": ...})` | 帶參數的 named URL |

## 11. 延伸：自訂 Author Model（老師加碼）

原本 `Post.author` 指向 Django 內建的 `auth.User`，這次改成**自己建一張表**來存作者資料。

### 為什麼要自己建一張表？

| 情境 | 適合的做法 |
|------|-----------|
| 作者 = 能登入後台的使用者 | 用 `auth.User`（內建帳密、權限） |
| 作者 = 純粹的「文章署名」，不需要登入 | 自建 `Author` Model（乾淨、欄位自由） |
| 作者需要額外欄位（如自我介紹、頭像）且也要能登入 | 用 Custom User（繼承 `AbstractUser`） |

這次用的是**第二種**：作者只是給文章掛個名字跟簡介，不涉及登入。

### models.py

```python
from django.db import models
from django.urls import reverse


class Author(models.Model):
    name = models.CharField(max_length=100)
    bio = models.TextField(blank=True)  # blank=True → 表單可以空白

    def __str__(self):
        return self.name


class Post(models.Model):
    title = models.CharField(max_length=200)
    author = models.ForeignKey(
        "Author",           # ← 從 "auth.User" 改成 "Author"
        on_delete=models.CASCADE,
    )
    body = models.TextField()

    def __str__(self):
        return self.title

    def get_absolute_url(self):
        return reverse("post_detail", kwargs={"pk": self.pk})
```

### ForeignKey 字串引用的小規則

| 寫法 | 說明 |
|------|------|
| `"Author"` | 同一個 app 內的 Model，直接寫類別名 |
| `"blog.Author"` | 跨 app 要加 app label（`blog.Author`） |
| `"auth.User"` | 內建 User |
| 直接 import 再傳類別 | 也可以，但字串寫法避免 circular import |

### admin.py

```python
from django.contrib import admin
from .models import Post, Author


class PostAdmin(admin.ModelAdmin):
    list_display = ("title", "author", "body")


class AuthorAdmin(admin.ModelAdmin):
    list_display = ("name", "bio")


admin.site.register(Post, PostAdmin)
admin.site.register(Author, AuthorAdmin)
```

→ 到 `/admin/` 就會看到 **Authors** 跟 **Posts** 兩張表，可以各自管理。

### Template 顯示作者

```html
<!-- home.html -->
<p class="post-author">作者：{{ post.author.name }}</p>

<!-- post_detail.html -->
<p class="post-author">作者：{{ post.author.name }}</p>
```

⚠️ 注意：以前是 `post.author.username`（因為 `auth.User` 的欄位叫 `username`），
現在我們的 `Author` 欄位叫 `name`，所以要改成 `post.author.name`。

### tests.py 也要跟著改

```python
from django.test import TestCase
from django.urls import reverse
from .models import Post, Author


class BlogTests(TestCase):
    @classmethod
    def setUpTestData(cls):
        cls.author = Author.objects.create(
            name="測試作者",
            bio="我是測試用的作者",
        )
        cls.post = Post.objects.create(
            title="A good title",
            body="Nice body content",
            author=cls.author,
        )

    def test_post_model(self):
        ...
        self.assertEqual(self.post.author.name, "測試作者")  # 不再是 .username
```

- 不需要 `get_user_model()` / `create_user()`（那是給 `auth.User` 用的）
- 直接 `Author.objects.create(...)` 就好，沒有密碼欄位

### 改 Model 後的 Migration 流程

因為從 `auth.User` 改成 `Author`，**外鍵的目標表換了**，舊資料會對不上。
這次因為還在學習階段，選擇**清空重來**：

```bash
# 1. 刪掉舊 db 跟舊 migration
rm db.sqlite3
rm blog/migrations/0001_initial.py

# 2. 重新產生 migration 並套用
python manage.py makemigrations blog
python manage.py migrate

# 3. 重建 superuser（admin 帳號也會一起被清掉）
python manage.py createsuperuser
```

> 💡 正式專案不能這樣做！會遺失所有資料。
> 正式流程要走**資料遷移（data migration）**：先加新表 → 搬資料 → 換 FK → 刪舊欄位，分好幾次 migration。

### 重點對照

```
舊版（用 auth.User）              新版（自建 Author）
─────────────────────────────    ─────────────────────────────
ForeignKey("auth.User")          ForeignKey("Author")
post.author.username             post.author.name
get_user_model().create_user     Author.objects.create
createsuperuser 順便建作者        admin 裡分開建 User 跟 Author
```

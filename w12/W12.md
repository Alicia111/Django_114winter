## 2. Stage 1：先讓首頁活起來

### 2.1 建立專案骨架

```bash
django-admin startproject django_project .
python manage.py startapp accounts
mkdir -p templates/registration
```

⚠️ **不要先跑 `migrate`！** 等 `CustomUser` 設好再一起 migrate，否則 auth 表會用內建 `User` 先建好，之後想換就翻車了。

### 2.2 CustomUser（`accounts/models.py`）

```python
from django.contrib.auth.models import AbstractUser
from django.db import models

class CustomUser(AbstractUser):
    age = models.PositiveIntegerField(null=True, blank=True)
```

- `AbstractUser` 已經包好 username / password / email / permission 等，**只要繼承再加欄位**。
- `age` 是這次的擴充欄位，Stage 2 之後會顯示在首頁。

### 2.3 設定 `settings.py`

```python
INSTALLED_APPS = [
    "accounts",              # 自己的 app 先放最上面
    "django.contrib.admin",
    "django.contrib.auth",
    "django.contrib.contenttypes",
    "django.contrib.sessions",
    "django.contrib.messages",
    "django.contrib.staticfiles",
]

TEMPLATES = [
    {
        ...
        "DIRS": [BASE_DIR / "templates"],   # 用「專案層級」的 templates 資料夾
        ...
    },
]

AUTH_USER_MODEL = "accounts.CustomUser"     # ⭐ 第一次 migrate 之前一定要設！
LOGIN_REDIRECT_URL = "home"
LOGOUT_REDIRECT_URL = "home"
```

### 2.4 第一個 View + Template

`templates/base.html`：放共用的版型（樣式、`<body>`、`{% block content %}`）。
`templates/home.html`：先簡單寫一段「Stage 1: Home page is running.」。

`accounts/views.py`：

`django_project/urls.py`：

## 3. Stage 2：註冊 / 登入 / 登出

### 3.1 註冊表單（`accounts/forms.py`）

```python
from django.contrib.auth.forms import UserCreationForm
from .models import CustomUser

class CustomUserCreationForm(UserCreationForm):
    class Meta:
        model = CustomUser
        fields = ("username", "email", "age")
```

> `UserCreationForm` 已經幫你包好密碼欄位（含確認密碼），
> 我們只要在 `Meta` 告訴它「model 是誰、要顯示哪幾個欄位」就好。

### 3.2 SignUpView（`accounts/views.py`）

```python
from django.urls import reverse_lazy
from django.views.generic import CreateView
from .forms import CustomUserCreationForm

class SignUpView(CreateView):
    form_class = CustomUserCreationForm
    success_url = reverse_lazy("login")
    template_name = "registration/signup.html"
```

- `CreateView` 一發現表單合法就會自動 `form.save()`，超省事。
- `reverse_lazy("login")` 是「**懶解析**」— 註冊成功後跳轉到 Django 內建 login 頁。

### 3.3 URL 路由

`accounts/urls.py`（**只放 signup**，login/logout 用內建的）：

```python
from django.urls import path
from .views import SignUpView

urlpatterns = [
    path("signup/", SignUpView.as_view(), name="signup"),
]
```

`django_project/urls.py`：

```python
from django.contrib import admin
from django.urls import include, path
from accounts.views import HomeView

urlpatterns = [
    path("admin/", admin.site.urls),
    path("accounts/", include("accounts.urls")),              # 我們的 signup
    path("accounts/", include("django.contrib.auth.urls")),   # ⭐ Django 內建的 login/logout
    path("", HomeView.as_view(), name="home"),
]
```

對應的 URL map：

| URL | 來源 |
|-----|------|
| `/` | `HomeView`（自己寫的） |
| `/accounts/signup/` | `SignUpView`（自己寫的） |
| `/accounts/login/` | Django 內建 `auth.urls` |
| `/accounts/logout/` | Django 內建 `auth.urls` |
| `/admin/` | Django admin |

### 3.4 登入 / 註冊頁 template

Django 內建的 `LoginView` 會自動找 `templates/registration/login.html`，命名要對到。

- `templates/registration/login.html`
- `templates/registration/signup.html`

兩個都用 `{% csrf_token %}` + `{{ form }}` 就好，畫面醜歸醜，但功能 100%。

### 3.5 改寫首頁，顯示登入狀態

`templates/home.html` 用 `{% if user.is_authenticated %}` 分兩種畫面：

- 已登入 → 顯示 `Hi {{ user.username }}! You are {{ user.age }} years old.` + 登出按鈕
- 未登入 → 顯示 `You are not logged in` + Log In / Sign Up 連結

> ⭐ 注意：**登出一定要用 POST**（Django 4.1+ 後 `LogoutView` 只接受 POST 了）
> 所以登出按鈕要包在 `<form method="post">` 裡 + `{% csrf_token %}`。

## 4. Stage 3：admin、Groups、角色卡片、seed 指令

這段是「**Extra extension**」— 原書沒這麼複雜，但加上去之後 admin 後台會更像真實場景。

### 4.1 admin 改造（`accounts/admin.py`）

要讓 manager 在 admin 後台能編輯 `age`、`is_staff`、Groups：

1. 在 `accounts/forms.py` 多加一個 `CustomUserChangeForm`（給 admin 編輯用）。
2. 寫 `CustomUserAdmin(UserAdmin)`，把 `age` 塞進 `fieldsets` 跟 `add_fieldsets`。
3. `admin.site.register(CustomUser, CustomUserAdmin)`。

```python
class CustomUserAdmin(UserAdmin):
    add_form = CustomUserCreationForm
    form = CustomUserChangeForm
    model = CustomUser
    list_display = ["username", "email", "age", "is_staff"]
    fieldsets = UserAdmin.fieldsets + ((None, {"fields": ("age",)}),)
    add_fieldsets = UserAdmin.add_fieldsets + ((None, {"fields": ("email", "age")}),)
```

### 4.2 ROLE_CARDS + HomeView 加 context

把 HomeView 改寫成會讀 `request.user.groups`，依角色發不同卡片：

```python
ROLE_CARDS = {
    "Student": "Student view: read course materials and check homework reminders.",
    "Teacher": "Teacher view: prepare lessons and review class progress.",
    "Assistant": "Assistant view: support students and collect common questions.",
}

class HomeView(TemplateView):
    template_name = "home.html"

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        user = self.request.user
        context["role_cards"] = []
        if user.is_authenticated:
            role_names = set(user.groups.values_list("name", flat=True))
            context["role_cards"] = [
                {"name": name, "message": message}
                for name, message in ROLE_CARDS.items()
                if name in role_names
            ]
        return context
```

`home.html` 用 `{% for role in role_cards %}` 渲染卡片，`{% empty %}` 顯示「還沒分配角色」的提示。

### 4.3 `seed_demo_roles` 管理指令

`accounts/management/commands/seed_demo_roles.py` — 跑一次就建好 4 個示範帳號 + 3 個 Group：

```bash
python manage.py seed_demo_roles
```

預期輸出：

```
Demo roles and users are ready.
manager / testpass123
student1 / testpass123
teacher1 / testpass123
assistant1 / testpass123
```

> 教室示範用，密碼 `testpass123` **千萬不要拿去做真網站**！

實作重點：

- `Group.objects.get_or_create(name=...)` → 重複跑不會炸。
- `user.set_password(...)` → 一定要用這個方法存密碼，不能直接賦值（否則會存明碼）。
- `user.groups.set([groups[group_name]])` → 設定使用者的 Group。

### 4.4 角色 → 首頁畫面對照

| 帳號 | 在首頁會看到 |
|------|--------------|
| `manager` (is_staff=True) | Manager view + 連到 admin 的連結 |
| `student1` (Group=Student) | Student 卡片 |
| `teacher1` (Group=Teacher) | Teacher 卡片 |
| `assistant1` (Group=Assistant) | Assistant 卡片 |
| 沒分 Group 的一般 user | 「No role assigned yet.」 |

## 6. 觀念地圖（Concept Summary）

這次學到的核心 Django 觀念：

- **`AUTH_USER_MODEL`** — 第一次 migrate 之前就要設好。
- **`UserCreationForm` + `CreateView`** — 註冊頁的標準組合。
- **`reverse_lazy("login")`** — class-based view 裡要用 lazy 版本，因為 URL config 還沒載入時就會被求值。
- **`django.contrib.auth.urls`** — 一行 `include(...)` 就把 login / logout / password reset 全部接好。
- **`user.is_authenticated`** — template 裡的標準判斷句。
- **CSRF + POST logout** — Django 4.1+ 後 logout 一定要 POST，不能再用 GET。
- **Django Groups** — 不用自己寫權限表，內建的 `Group` 就能做角色分類。
- **Custom management command** — 把 setup 流程包成 `python manage.py <cmd>`，教室示範或測試環境超方便。

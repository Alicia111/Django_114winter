## 2. Project-Level Templates（專案層 Templates）

W3 的 template 放在 `app/templates/app/`（app-level，需要命名空間）。
W4 改用 **project-level templates**：在專案根目錄建立單一 `templates/` 資料夾，所有 app 的 template 都放這裡。

```bash
mkdir templates
```

優點：所有 template 集中管理，更容易找到與維護。
需要在 `settings.py` 的 `TEMPLATES["DIRS"]` 告訴 Django 去哪裡找。

## 3. Template Context、Tags、Filters

### 常用 Tags

| 語法 | 說明 |
|------|------|
| `{% now "DATETIME_FORMAT" %}` | 顯示目前日期時間 |
| `{% for item in list %}...{% endfor %}` | 迴圈 |
| `{% comment %}...{% endcomment %}` | 註解 |

### 常用 Filters

| 語法 | 說明 |
|------|------|
| `{{ list\|length }}` | 取得長度 |
| `{{ str\|title }}` | 每個單字首字大寫 |
| `{{ str\|upper }}` | 全大寫 |

```python
# pages/views.py（FBV）
def home_page_view(request):
    context = {
        "inventory_list": ["Widget 1", "Widget 2", "Widget 3"],
        "greeting": "THAnk you FOR visitING.",
    }
    return render(request, "home.html", context)
```

```html
<!-- templates/home.html -->
<p>{% now "DATETIME_FORMAT" %}</p>
<p>{{ inventory_list|length }} items</p>
{% for item in inventory_list %}
  <li>{{ item }}</li>
{% endfor %}
<p>{{ greeting|title }}</p>
```

## 4. Class-Based Views（CBV）

Django 有三種 View 寫法：

| 種類 | 說明 |
|------|------|
| Function-Based View (FBV) | 傳統函式，直觀易懂 |
| Class-Based View (CBV) | 使用 Python class，可繼承擴展 |
| Generic Class-Based View | Django 內建通用 CBV（如 `TemplateView`） |

### TemplateView

最簡單的 CBV，只需指定 `template_name`：

```python
# pages/views.py
from django.views.generic import TemplateView

class AboutPageView(TemplateView):
    template_name = "about.html"
```

URL 設定時要呼叫 `.as_view()`：

```python
# pages/urls.py
path("about/", AboutPageView.as_view(), name="about"),
```

### get_context_data()

CBV 中傳遞 context 的標準方法，要呼叫 `super()` 取得既有 context 再加入新的 key：

```python
class AboutPageView(TemplateView):
    template_name = "about.html"

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        context["contact_address"] = "123 Main Street"
        context["phone_number"] = "555-555-5555"
        return context
```

## 5. Template Inheritance（模板繼承）

### 概念

建立一個 `base.html` 父模板，定義所有頁面共用的 HTML（如導覽列）。
子模板繼承父模板，只需填入各自的內容區塊。

### base.html

```html
<!-- templates/base.html -->
<header>
  <a href="{% url 'home' %}">Home</a> |
  <a href="{% url 'about' %}">About</a>
</header>

{% block content %}{% endblock %}
```

- `{% block content %}{% endblock %}`：定義「可被子模板覆寫」的區域

### 子模板

```html
<!-- templates/home.html -->
{% extends "base.html" %}

{% block content %}
<h1>Company Homepage</h1>
...
{% endblock %}
```

- `{% extends "base.html" %}`：宣告繼承哪個父模板（必須在第一行）
- 子模板只需填入 `{% block content %}` 區塊內的內容

## 6. Named URLs

在 `path()` 加上 `name` 參數，之後 template 和測試都能用名稱取代硬編碼路徑：

```python
# pages/urls.py
urlpatterns = [
    path("about/", AboutPageView.as_view(), name="about"),
    path("", home_page_view, name="home"),
]
```

Template 中使用 `{% url '名稱' %}`：

```html
<a href="{% url 'home' %}">Home</a>
<a href="{% url 'about' %}">About</a>
```

好處：URL 路徑只需在 `urls.py` 改一個地方，template 自動更新。

## 7. Testing

### 本週新增的測試方法

| 方法 | 說明 |
|------|------|
| `reverse("name")` | 透過 URL name 取得 URL 路徑，避免硬編碼 |
| `assertTemplateUsed(response, "home.html")` | 確認使用了正確的 template |
| `assertContains(response, "<h1>...</h1>")` | 確認 HTML 內容存在 |

### 完整測試範例

```python
# pages/tests.py
from django.test import SimpleTestCase
from django.urls import reverse

class HomepageTests(SimpleTestCase):
    def test_url_exists_at_correct_location(self):
        response = self.client.get("/")
        self.assertEqual(response.status_code, 200)

    def test_url_available_by_name(self):
        response = self.client.get(reverse("home"))
        self.assertEqual(response.status_code, 200)

    def test_template_name_correct(self):
        response = self.client.get(reverse("home"))
        self.assertTemplateUsed(response, "home.html")

    def test_template_content(self):
        response = self.client.get(reverse("home"))
        self.assertContains(response, "<h1>Company Homepage</h1>")
```

## 重點整理

```
FBV vs CBV：
  FBV → 函式，直觀，適合簡單邏輯
  CBV (TemplateView) → 類別，可繼承，適合複雜或重複的模式

Template 組織：
  W3：app/templates/app/template.html（需要命名空間）
  W4：templates/template.html（project-level，settings 加 DIRS）

Template Inheritance：
  base.html（父）→ {% block content %}{% endblock %}
  child.html（子）→ {% extends "base.html" %} + {% block content %}

Named URLs：
  urls.py  → path("", view, name="home")
  template → {% url 'home' %}
  tests    → reverse("home")
```

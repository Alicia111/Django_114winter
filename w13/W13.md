## 4. Stage 2：Article + Comment models（Ch13 + Ch15 的 model）

### 4.1 `articles/models.py`

```python
from django.conf import settings
from django.db import models
from django.urls import reverse


class Article(models.Model):
    title = models.CharField(max_length=255)
    body = models.TextField()
    date = models.DateTimeField(auto_now_add=True)
    author = models.ForeignKey(
        settings.AUTH_USER_MODEL,    # ⭐ 不要直接 import CustomUser
        on_delete=models.CASCADE,
    )

    def __str__(self):
        return self.title

    def get_absolute_url(self):
        return reverse("article_detail", kwargs={"pk": self.pk})


class Comment(models.Model):
    article = models.ForeignKey(Article, on_delete=models.CASCADE)
    comment = models.CharField(max_length=140)
    author = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        on_delete=models.CASCADE,
    )

    def __str__(self):
        return self.comment

    def get_absolute_url(self):
        return reverse("article_list")
```

⭐ **觀念複習**：在 model 檔案內，author 一律用 `settings.AUTH_USER_MODEL`（字串）；
不要在 model 檔 import 具體的 User class，否則改 User Model 時會炸。

### 4.2 `articles/admin.py`（Ch15 的 `TabularInline`）

```python
class CommentInline(admin.TabularInline):
```

## 5. Stage 3：Article Views + 權限（Ch13 + Ch14 的核心）

### 5.1 CommentForm（Ch15）

`articles/forms.py`：

```python
from django import forms
from .models import Comment


class CommentForm(forms.ModelForm):
    class Meta:
        model = Comment
        fields = ("comment",)
```

> ⭐ 只暴露 `comment` 欄位，**author 跟 article 在 view 裡面自己塞**。
> 不能讓使用者在表單上選 author，否則就能假冒別人留言了。

### 5.2 `articles/views.py` — 整章最關鍵

我們有 5 個 view，分四類：

| View | Mixins | 功能 |
|------|--------|------|
| `ArticleListView` | `LoginRequiredMixin` + `ListView` | 文章列表，要登入 |
| `ArticleDetailView` | 自己包裝的 `View`，把 GET / POST 拆開 | 詳情頁 + 留言表單 |
| `ArticleCreateView` | `LoginRequiredMixin` + `CreateView` | 新文章，author 自動帶入 |
| `ArticleUpdateView` | `LoginRequiredMixin` + `UserPassesTestMixin` + `UpdateView` | 只有作者能編輯 |
| `ArticleDeleteView` | `LoginRequiredMixin` + `UserPassesTestMixin` + `DeleteView` | 只有作者能刪除 |

#### 重點 1：CreateView 自動帶入 author（Ch14）

```python
class ArticleCreateView(LoginRequiredMixin, CreateView):
    model = Article
    fields = ("title", "body")          # 注意沒有 author！
    template_name = "article_new.html"

    def form_valid(self, form):
        form.instance.author = self.request.user   # ⭐ 自動填作者
        return super().form_valid(form)
```

> Ch11 還要手動選 author（很怪），Ch14 改成 `form_valid` 自動帶 `request.user`，
> 表單就乾淨多了 `( •̀ ω •́ )y`

#### 重點 2：UserPassesTestMixin（Ch14）

```python
class ArticleUpdateView(LoginRequiredMixin, UserPassesTestMixin, UpdateView):
    model = Article
    fields = ("title", "body")
    template_name = "article_edit.html"

    def test_func(self):
        obj = self.get_object()
        return obj.author == self.request.user
```

- `test_func` 回 `True` → 通過；回 `False` → **403 Forbidden**。
- Mixin 順序：**`LoginRequiredMixin` 一定要放前面**，先確認登入，再判作者。

#### 重點 3：CommentGet + CommentPost 拆開（Ch15 的進階技巧）

留言頁面同時要做兩件事：
- **GET** → 顯示文章 + 空白留言表單
- **POST** → 存留言、跳回詳情頁

直接寫一個 View 兩件都做會很亂，書本的解法是 **拆成兩個 view，再包一層 wrapper**：

```python
class CommentGet(DetailView):           # GET 用，只顯示
    model = Article
    template_name = "article_detail.html"

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        context["form"] = CommentForm()
        return context


class CommentPost(SingleObjectMixin, FormView):    # POST 用，處理表單
    model = Article
    form_class = CommentForm
    template_name = "article_detail.html"

    def post(self, request, *args, **kwargs):
        self.object = self.get_object()
        return super().post(request, *args, **kwargs)

    def form_valid(self, form):
        comment = form.save(commit=False)
        comment.article = self.object
        comment.author = self.request.user
        comment.save()
        return super().form_valid(form)

    def get_success_url(self):
        article = self.object
        return reverse("article_detail", kwargs={"pk": article.pk})


class ArticleDetailView(LoginRequiredMixin, View):     # wrapper
    def get(self, request, *args, **kwargs):
        view = CommentGet.as_view()
        return view(request, *args, **kwargs)

    def post(self, request, *args, **kwargs):
        view = CommentPost.as_view()
        return view(request, *args, **kwargs)
```

| 角色 | 負責的事 |
|------|----------|
| `CommentGet` | DetailView，幫 template 多塞一個空白 `form` |
| `CommentPost` | `SingleObjectMixin`（拿 article）+ `FormView`（處理表單） |
| `ArticleDetailView` | `LoginRequiredMixin` + `View`：GET 路由到 Get，POST 路由到 Post |

> 一開始看會覺得「為什麼這麼複雜」，但這就是 Django generic view 的標準拆法 —
> 一個 view 一件事，職責清楚、好測試 `(｀・ω・´)`

## 6. Stage 4：CRUD Templates

### 6.1 改寫 `base.html`，nav 加上 Articles 連結

登入後 nav 多出 `Articles` 和 `New Article` 兩個入口，未登入則只看到 Log In / Sign Up。

### 6.2 `article_list.html`

```html
{% for article in article_list %}
  <article>
    <h2><a href="{{ article.get_absolute_url }}">{{ article.title }}</a></h2>
    <p>by {{ article.author }} | {{ article.date }}</p>
    <p>{{ article.body }}</p>
    {% if article.author.pk == request.user.pk %}
      <p>
        <a href="{% url 'article_edit' article.pk %}">Edit</a> |
        <a href="{% url 'article_delete' article.pk %}">Delete</a>
      </p>
    {% endif %}
    ...
    {% for comment in article.comment_set.all %}
      <p><strong>{{ comment.author }}</strong> &middot; {{ comment }}</p>
    {% empty %}
      <p>No comments yet.</p>
    {% endfor %}
  </article>
{% empty %}
  <p>No articles yet.</p>
{% endfor %}
```

⭐ **`{% if article.author.pk == request.user.pk %}`**：
非作者連 Edit/Delete 連結都看不到，這是 UI 層的權限保護。
真正的「保護」還是要在 view 用 `UserPassesTestMixin`，否則手動打 URL 還是闖得進去。

### 6.3 `article_detail.html`

詳情頁的特色是「**同一個 URL 同時是 GET 顯示頁 + POST 提交頁**」：

```html
<form action="" method="post">
  {% csrf_token %}
  {{ form.as_p }}
  <button type="submit">Save Comment</button>
</form>
```

`action=""` = submit 回到當前 URL，剛好被 `ArticleDetailView.post()` 路由到 `CommentPost`。

### 6.4 `article_new.html` / `article_edit.html` / `article_delete.html`

跟 W8 的 blog CRUD 一樣的結構，只是改成 `article_*`。

## 7. Stage 5：`seed_demo_articles` 指令

`articles/management/commands/seed_demo_articles.py`：

```python
class Command(BaseCommand):
    help = "Create demo articles and comments for Ch13-15."

    def handle(self, *args, **options):
        call_command("seed_demo_roles")    # 先確保帳號跟 Group 都建好
        user_model = get_user_model()
        student = user_model.objects.get(username="student1")
        teacher = user_model.objects.get(username="teacher1")
        assistant = user_model.objects.get(username="assistant1")

        first_article, _ = Article.objects.get_or_create(
            title="Campus News",
            defaults={"body": "...", "author": teacher},
        )
        second_article, _ = Article.objects.get_or_create(
            title="Student Report",
            defaults={"body": "...", "author": student},
        )
        Comment.objects.get_or_create(
            article=first_article, comment="...", author=student,
        )
        Comment.objects.get_or_create(
            article=second_article, comment="...", author=assistant,
        )
```

| 小技巧 | 為什麼 |
|--------|--------|
| `call_command("seed_demo_roles")` | 教學示範時，一行就把全部資料弄好（帳號 + 文章 + 留言） |
| `get_or_create(...)` | 重複跑不會報錯，也不會建出重複資料 |

## 9. 觀念地圖 — 三章合起來學到了什麼

### Ch13（Articles app）
- `ForeignKey(settings.AUTH_USER_MODEL, on_delete=CASCADE)` 接 author
- `ListView` / `DetailView` / `CreateView` / `UpdateView` / `DeleteView` 五件組
- `get_absolute_url()` + `reverse(...)` 讓 model 自帶 URL

### Ch14（Permissions）
- **`LoginRequiredMixin`** — 要登入才能進
- **`UserPassesTestMixin`** + `test_func()` — 自訂條件，回 False 直接 403
- **Mixin 順序**：`LoginRequiredMixin` → `UserPassesTestMixin` → 主 view
- **`form_valid(self, form)`** 改 `form.instance.author = self.request.user` 自動帶作者
- 雙重保護：template 隱藏連結 + view 真的擋

### Ch15（Comments）
- `Comment(article=FK, comment=Char, author=FK)` 一對多反向 `comment_set`
- **`TabularInline`** 把 Comment 嵌進 Article admin 頁
- **`ModelForm`** 只暴露安全欄位（author/article 由 view 注入）
- **`SingleObjectMixin` + `FormView`** — 表單處理跟 DetailView 拆開
- 一個 URL 同時對 GET / POST，用 wrapper view 分流

## 12. 常見小坑 `(^_^;)`

| 症狀 | 原因 / 修法 |
|------|-------------|
| `ModuleNotFoundError: No module named 'articles.urls'` | 還沒建 `articles/urls.py` 就在 `django_project/urls.py` include 了 |
| `RelatedObjectDoesNotExist: Article has no author.` | CreateView 漏寫 `form.instance.author = self.request.user` |
| 非作者也能進編輯頁 | view 沒掛 `UserPassesTestMixin`（template 隱藏連結只是 UI 層） |
| 留言送出後沒跳轉 | `CommentPost.get_success_url` 寫錯 |
| `LogoutView` 報 405 | Django 4.1+ 後 logout 必須 POST，不能 GET |
| `python manage.py test` 跑 0 個 | tests.py 被覆寫成預設模板，記得貼上實際測試 |

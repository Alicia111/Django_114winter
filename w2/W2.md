
## 4. Django 專案初始化

```bash
django-admin startproject django_project .
```

> 結尾的 `.` 很重要，代表在**當前目錄**建立專案，不會多一層資料夾。

產生的目錄結構：

```
django_project/
 ├── django_project/
 │   ├── __init__.py
 │   ├── asgi.py
 │   ├── settings.py
 │   ├── urls.py
 │   └── wsgi.py
 └── manage.py
```

| 檔案 | 說明 |
|------|------|
| `__init__.py` | 讓 Python 把這個資料夾當作 package |
| `asgi.py` | ASGI (非同步) 伺服器入口 |
| `settings.py` | 專案設定（資料庫、App、靜態檔等） |
| `urls.py` | URL 路由設定 |
| `wsgi.py` | WSGI (同步) 伺服器入口 |
| `manage.py` | Django 命令列工具 |

## 5. 資料庫遷移 & 啟動伺服器

```bash
# 初始化資料庫 (建立預設的資料表)
python manage.py migrate

# 啟動開發伺服器
python manage.py runserver

# 也可以指定 port
python manage.py runserver 8080
```

開啟瀏覽器到 `http://127.0.0.1:8000/` 看到 Django 的歡迎頁面就成功了。

## 6. 建立第一個 App - Hello World

### 6-1. 建立 App

```bash
python manage.py startapp pages
```

產生的 `pages/` 結構：

```
pages/
 ├── __init__.py
 ├── admin.py
 ├── apps.py
 ├── migrations/
 │   └── __init__.py
 ├── models.py
 ├── tests.py
 └── views.py
```

### 6-2. 註冊 App

在 `django_project/settings.py` 的 `INSTALLED_APPS` 加入 `"pages"`：

```python
INSTALLED_APPS = [
    "django.contrib.admin",
    "django.contrib.auth",
    "django.contrib.contenttypes",
    "django.contrib.sessions",
    "django.contrib.messages",
    "django.contrib.staticfiles",
    "pages",  # <-- 新增
]
```

### 6-3. 寫 View

在 `pages/views.py`：

```python
from django.http import HttpResponse

def home_page_view(request):
    return HttpResponse("Hello, World!")
```

### 6-4. 設定 App 的 URL

新建 `pages/urls.py`：

```python
from django.urls import path
from .views import home_page_view

urlpatterns = [
    path("", home_page_view),
]
```

### 6-5. 把 App URL 接到專案

在 `django_project/urls.py` 加入 `include`：

```python
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path("admin/", admin.site.urls),
    path("", include("pages.urls")),  # <-- 新增
]
```

開啟 `http://127.0.0.1:8000/` 就能看到 **Hello, World!**

## 重點整理

```
Django 的 Request/Response 流程：

瀏覽器 Request
    ↓
urls.py (路由比對)
    ↓
views.py (處理邏輯)
    ↓
HttpResponse (回傳給瀏覽器)
```

- **Project vs App**：一個 Django Project 可以包含多個 App，每個 App 負責一個功能模組
- **`include()`**：讓每個 App 管理自己的 URL，保持程式碼分離
- **`migrate`**：將 Model 定義同步到資料庫（即使沒寫自訂 Model，Django 內建的 auth、session 等也需要 migrate）
